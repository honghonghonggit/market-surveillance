# 프로젝트 기획서: 이상거래 탐지(시장감시) 엔진

## 목적
실시간 주문 스트림에서 시세조종 패턴(스푸핑·레이어링·워시트레이딩)을 탐지하는 시스템을 구현한다. 코스콤이 KRX를 위해 운영하는 시장감시시스템(CAMS)의 핵심 개념을 이해하고, 시세조종 탐지의 원리와 ML 평가 방법론을 직접 구현해보는 것이 목표다. 100% 합성 데이터를 사용하며, 실제 종목·거래소·투자자와 무관함을 명시한다.

## 차별화 포인트 (가장 중요)
대부분의 학부생 ML 프로젝트는 "정확도 95%"로 끝나지만 그 숫자가 신뢰할 만한지 검증할 방법이 없다. 이 프로젝트는 합성 데이터에 조작 패턴을 직접 주입해 ground truth(정답 라벨)를 확보한 상태에서 모델을 평가하므로, 정밀도·재현율·혼동행렬·ROC 커브가 진짜 의미를 갖는다. RA-Testbed의 lookahead bias 방지가 핵심이었던 것처럼, 이 프로젝트는 "평가 방법론의 엄밀함"이 핵심이다.

## 탐지 대상 패턴
- 스푸핑(Spoofing): 대량 주문으로 가격을 유인한 뒤 체결 직전 취소. 비정상적으로 높은 주문 취소율이 특징.
- 레이어링(Layering): 여러 가격 레벨에 주문을 층층이 쌓아 호가창을 왜곡. 다중 레벨 동시 주문 패턴이 특징.
- 워시트레이딩(Wash Trading): 자기매매로 허위 거래량 생성. 짧은 시간 내 동일 가격 매수/매도 반복이 특징.

## 전체 흐름
합성 데이터 생성기(패턴 주입 + ground truth 라벨) → Kafka 프로듀서 → Kafka 토픽 → 컨슈머 → 피처 엔지니어링 → 탐지 모델 → 평가(정밀도/재현율/ROC) → Streamlit 대시보드

## 기술 스택
- Python 3.12, pandas, numpy
- Kafka (Docker 로컬 구동), kafka-python/confluent-kafka
- scikit-learn (룰 기반 + ML 분류기), 필요시 XGBoost
- Streamlit + Plotly (대시보드)
- pytest

## 단계별 범위

### Phase 1 — MVP (먼저 끝낼 것)
- 합성 주문 데이터 생성기: 정상 주문 흐름(랜덤워크 기반) 생성
- 패턴 주입: 스푸핑·레이어링·워시트레이딩 중 최소 1~2종을 주입하고, 각 주입 구간에 ground truth 라벨 부착
- 기본 피처 엔지니어링: 주문 취소율, 주문-체결 비율 등
- 룰 기반 탐지(임계값) + 정밀도·재현율·혼동행렬 평가
- (Kafka 없이) 인메모리 스트림으로 먼저 동작 검증

### Phase 2 — 차별화 기능 (Phase1 완료 후)
- Kafka 스트리밍 파이프라인 도입(프로듀서/컨슈머 분리, 실시간 처리)
- 패턴 3종 모두 주입 + 정교한 피처(가격충격, 다중 레벨 주문 밀도, 시간 윈도우 기반 지표)
- ML 분류기(scikit-learn) 도입 + 룰 기반과 성능 비교
- ROC 커브, PR 커브, 임계값에 따른 정밀도-재현율 트레이드오프 시각화
- Streamlit 대시보드: 실시간 주문 흐름 + 탐지 알림 + 평가 지표 패널

### Phase 3 — 스트레치 (시간 남으면)
- 패턴별 탐지 정확도 분해(어떤 패턴을 잘 잡고 어떤 걸 놓치는지)
- 정상 거래를 오탐(false positive)하는 케이스 분석
- 탐지 지연(latency) 측정 — 패턴 발생부터 탐지까지 시간
- 배포(Kafka 포함이라 배포 환경 신중히 결정)

## 평가 방법론 (이 프로젝트의 핵심 — 반드시 문서화)
- ground truth: 패턴을 주입한 시점/주문을 정답 라벨로 보유
- 정밀도(Precision): 탐지한 것 중 진짜 조작인 비율 → 오탐을 줄이는 게 중요한 이유
- 재현율(Recall): 진짜 조작 중 탐지한 비율 → 놓친 조작을 줄이는 게 중요한 이유
- 혼동행렬, ROC/PR 커브로 임계값 트레이드오프 시각화
- "정확도(accuracy) 하나로 평가하면 안 되는 이유"(불균형 데이터: 조작은 전체의 극소수)를 명시

## 주의사항 (법적/윤리)
- 100% 합성 데이터. 실제 종목·거래소·투자자를 지목하지 않는다.
- "코스콤 CAMS를 그대로 재현"이 아니라 "개념을 이해하고 핵심 원리를 구현"으로 서술.
- 교육·포트폴리오 목적임을 명시.

## 포트폴리오 활용
- 코스콤 시장감시(CAMS) 사업과 직접 연결되는 스토리. 면접에서 "CAMS를 보고 착안했다"고 설명 가능.
- 면접 대비: "왜 accuracy가 아니라 정밀도/재현율을 봤는지", "ground truth를 어떻게 확보했는지", "스푸핑을 어떤 피처로 잡았는지", "룰 기반과 ML 중 왜 그걸 택했는지" 질문에 답할 수 있도록 설계 과정을 기록.
